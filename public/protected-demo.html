🎓 What is a Protected Route?

A protected route is a web page or API endpoint that requires authentication to access. It's like having a bouncer at the door of an exclusive club.

  • Users must prove their identity (authenticate) before entering
  • Without valid credentials, access is denied
  • The page checks authentication status before showing sensitive content
  • Common in apps for user dashboards, account settings, private data, etc.
📝 Basic Authentication Check

This is how you check if a user is authenticated on the frontend:

// Check if user has a valid JWT token function isAuthenticated() { const token = localStorage.getItem('jwtToken'); if (!token) { return false; // No token found } // Optionally: verify token isn't expired try { const payload = JSON.parse(atob(token.split('.')[1])); const currentTime = Math.floor(Date.now() / 1000); return payload.exp > currentTime; // Check expiration } catch (error) { return false; // Invalid token format } }
🔄 Automatic Redirect Example

Many apps automatically redirect unauthenticated users:

// Protect the entire page with automatic redirect window.addEventListener('load', function() { const token = localStorage.getItem('jwtToken'); if (!token) { // Save the current page URL for after login localStorage.setItem('redirectAfterLogin', window.location.href); // Redirect to login page window.location.href = '/auth.html'; return; } // User is authenticated, show protected content showProtectedContent(); });
